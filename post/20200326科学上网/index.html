准备

一台完美的跳转机

对于性能没有太大的需求，延迟越小越好，价格越低越好。 这里不推荐了，避免不必要的麻烦。

系统的选择

一般来说价格便宜的只有选linux系统，centos，redhat都无所谓。

一些基础知识

linux命令，docker命令，一些网络基础。

安装docker

首先默认你已经有一台完美的跳转机，并且拿到了他的最高权限root。 那么我们开始部署docker。

docker对于centos的版本是有要求的，具体可以去docker网站上看下，一般centos7以上基本没什么问题。

需要删除之前的遗留的老版本。

用docker来启动ss

直接一条命令搞定，docker会自动下载image并且启动。

1

docker run -dt --name ss-server -p 6445:6445 mritd/shadowsocks -s "-s 0.0.0.0 -p 6445 -m aes-256-cfb -k 123456test --fast-open" --restart=always

这里来解释下参数的含义： -t 为容器重新分配一个伪输入终端，通常与 -i 同时使用；

-d 后台运行容器，并返回容器ID；

–name 为容器起名;

-p 6445：6445 把容器上的6445端口映射到宿主机的6445上[宿主机端口：容器端口];

mritd/shadowsocks ss的镜像名;

-s 跟一些ss镜像特有的参数;

-s 0.0.0.0 限制源地址，0.0.0.0即是允许所有;

-p 6445 容器内部使用6445端口，这个可以自定义，这里变了之前的-p参数也要相应变化;

-m aes-256-cfb 使用加密的方式;

-k <password> 配置密钥;

–restart=always 容器退出后总是启动;

现在你的ss已经开始服务了，但是大概率这个时候还是连不上的 因为centos7默认开启firewalld，默认没有开放6445端口。

关闭防火墙

1
2

systemctl stop firewalld.service
systemctl disable firewalld.service

科学上网

下载ss客户端，填上相应的信息，开始上网吧
